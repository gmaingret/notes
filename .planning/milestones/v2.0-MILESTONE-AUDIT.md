### Integration Issue

**refreshToken cookie not cleared on logout (Medium)**

`AuthRepositoryImpl.logout()` calls `tokenStore.clearAll()` (clears access token + user email) but does NOT call `DataStoreCookieJar.clearAll()`. The refreshToken httpOnly cookie persists in a separate DataStore after logout. On next cold start, `SplashViewModel` → `CheckAuthUseCase` → `POST /api/auth/refresh` may silently re-authenticate the previous user.

- **File:** `android/.../data/repository/AuthRepositoryImpl.kt` lines 71-78
- **Fix:** Inject `DataStoreCookieJar` and call `cookieJar.clearAll()` in `logout()`
- **Affected:** AUTH-05, AUTH-06
