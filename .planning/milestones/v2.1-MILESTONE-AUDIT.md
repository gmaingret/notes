### Integration Findings (Non-Critical)

1. **INT-01: Unreachable WidgetEntryPoint.createBulletUseCase()** (ACT-01)
   - `AddBulletActivity` uses `@AndroidEntryPoint`/`@Inject` directly, not `EntryPointAccessors`
   - The method on `WidgetEntryPoint` has no production caller — dead interface surface
   - Severity: Cosmetic (future-proofing artifact)

2. **INT-02: clearAll() on single widget deletion** (SETUP-01, SYNC-02)
   - `NotesWidgetReceiver.onDeleted` wipes all widget data and cancels WorkManager sync unconditionally
   - Multi-widget users would lose the surviving widget's configuration
   - Severity: Behavioral gap in multi-widget scenario (single-widget use is correct)
   - Listed as future scope: WIDG-01 (multiple widget instances) in REQUIREMENTS.md

3. **INT-03: Eventual widget recovery after re-login** (SYNC-03)
   - After re-login, `onResume` re-pushes stale SESSION_EXPIRED cache
   - Widget recovers on next WorkManager tick (up to 15 min) or opening pinned document
   - Severity: UX cosmetic — documented behavior in Phase 14 verification
