    2. Inject via constructor: `BulletRepository`, `DocumentRepository`, `WidgetStateStore` (the Hilt-provided singleton — add it to WidgetModule if not already provided, or use WidgetEntryPoint pattern). Since WidgetStateStore is a manual singleton with `getInstance()`, create a `@Provides` method in WidgetModule that calls `WidgetStateStore.getInstance(context)` using `@ApplicationContext context`.
       - **Actually:** The worker runs in-process so it can use Hilt singletons. But WidgetStateStore uses a manual singleton pattern. The simplest approach: inject `@ApplicationContext context: Context` and call `WidgetStateStore.getInstance(context)` inside `doWork()`. This avoids needing to change the DI setup. The research already confirmed WidgetModule provides WidgetStateStore via @Provides.
       - Check if WidgetModule already provides WidgetStateStore. If yes, inject directly. If not, just call getInstance(applicationContext) in doWork().
    3. `override suspend fun doWork(): Result`:
       a. Get docId from `WidgetStateStore.getFirstDocumentId()`. If null, return `Result.success()` (widget not configured).
       b. Fetch documents to validate docId exists and get title. On failure: if auth error, write `SESSION_EXPIRED` + `updateAll()` + return success. If other error, keep cached data (don't overwrite), return success.
       c. Find document by docId. If not found, write `DOCUMENT_NOT_FOUND` + `updateAll()` + return success.
       d. Fetch bullets for docId. On failure: same error handling as step b.
       e. Filter root-level bullets (`parentId == null`), take first 50, map to `WidgetBullet` using `stripMarkdownSyntax`.
       f. Write bullets to `WidgetStateStore.saveBullets()`.
       g. Write display state: `CONTENT` if non-empty, `EMPTY` if empty.
       h. Call `NotesWidget().updateAll(applicationContext)`.
       i. Return `Result.success()`. Always return success to keep the periodic schedule alive.
    4. Add private `isAuthError(e: Throwable?)` helper (same pattern as NotesWidget.kt).
