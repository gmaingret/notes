  <name>Task 1: undoService — undo stack management</name>
  <files>server/src/services/undoService.ts</files>
  <behavior>
    - recordUndoEvent: inserts undo_event row, advances cursor seq, truncates redo stack (events > current seq), drops oldest when 51st event recorded (FIFO)
    - undo: applies inverseOp of event at currentSeq, decrements cursor; returns { canUndo, canRedo }
    - redo: applies forwardOp of event at currentSeq+1, increments cursor; returns { canUndo, canRedo }
    - getStatus: returns { canUndo: currentSeq > minSeq, canRedo: events exist above cursor }
  </behavior>
  <action>
Create `server/src/services/undoService.ts`.

Use the exact pattern from the research document (Pattern 4 — Undo Wrapping). Key implementation details:

**recordUndoEvent(dbInstance, userId, eventType, forwardOp, inverseOp)**:
1. Fetch current cursor (or 0 if none)
2. newSeq = currentSeq + 1
3. In one transaction: DELETE events WHERE userId=userId AND seq > currentSeq (truncate redo); DELETE events WHERE userId=userId AND seq < newSeq - 50 (FIFO cap); INSERT new event with seq=newSeq, schemaVersion=1; UPSERT cursor to currentSeq=newSeq

**undo(dbInstance, userId)**:
1. Fetch cursor; if currentSeq === 0, return { canUndo: false, canRedo: ... }
2. Fetch event at currentSeq
3. Parse inverseOp and execute it (see op format below)
4. Decrement cursor: UPDATE undo_cursors SET currentSeq = currentSeq - 1
5. Return getStatus(dbInstance, userId)

**redo(dbInstance, userId)**:
1. Fetch cursor; check if event exists at currentSeq + 1
2. Fetch that event; parse forwardOp; execute it
3. Increment cursor
4. Return getStatus(dbInstance, userId)

**Op format** (jsonb stored and parsed): Each op is a discriminated union by `type`:
- `{ type: 'create_bullet', bullet: BulletRow }` — inverse of delete
- `{ type: 'delete_bullet', id: string }` — sets deletedAt = now()
- `{ type: 'restore_bullet', id: string }` — sets deletedAt = null
- `{ type: 'update_bullet', id: string, fields: Partial<BulletRow> }` — patches specific columns
- `{ type: 'move_bullet', id: string, parentId: string|null, position: number }` — sets parent+position

The undo service executes ops directly via Drizzle. It does NOT call bulletService (prevents circular dependency). Import db from `'../../db/index.js'` as default dbInstance parameter.

Export: `recordUndoEvent`, `undo`, `redo`, `getStatus`. Types: export `UndoOp` discriminated union.

**computeBulletInsertPosition(dbInstance, documentId, parentId, afterId)**:
Copy the algorithm from `computeDocumentInsertPosition` in documentService.ts. Query siblings WHERE documentId=$doc AND parentId matches (use `isNull` for null parentId) AND deletedAt IS NULL, ORDER BY position ASC. Compute midpoint or tail +1.0.

**createBullet(userId, { documentId, parentId, afterId, content })**:
```
position = await computeBulletInsertPosition(db, documentId, parentId, afterId)
const id = uuidv4()
await db.transaction(async tx => {
  await tx.insert(bullets).values({ id, userId, documentId, parentId: parentId ?? null, content, position, isComplete: false, isCollapsed: false })
  await recordUndoEvent(tx, userId, 'create_bullet',
    { type: 'create_bullet', id, documentId, parentId, content, position },  // forwardOp (re-create if undone)
    { type: 'restore_bullet_delete', id }  // inverseOp = soft delete
  )
})
return created bullet
```

**indentBullet(userId, bulletId)**:
1. Load bullet; get its siblings in position order
2. Find this bullet's index; if index === 0, return (no-op, first sibling)
3. previousSibling = siblings[index - 1]
4. newPosition = compute as last child of previousSibling
5. Snapshot old state for inverse_op
6. In transaction: UPDATE bullet SET parentId=previousSibling.id, position=newPosition; recordUndoEvent with inverse_op restoring old parentId+position

**outdentBullet(userId, bulletId)**:
1. Load bullet; if parentId is null, return (no-op, root level)
2. Load parent bullet; get parent's siblings
3. newPosition = midpoint after parent's current position among parent's siblings
4. In transaction: UPDATE bullet SET parentId=parent.parentId, position=newPosition; recordUndoEvent

**moveBullet(userId, bulletId, { newParentId, afterId })**:
1. Cycle guard: load all descendants of bulletId; if newParentId is in that set, throw error
2. Compute newPosition
3. In transaction: UPDATE bullet SET parentId=newParentId, position=newPosition; recordUndoEvent

**softDeleteBullet(userId, bulletId)**:
In transaction: UPDATE SET deletedAt=new Date(); recordUndoEvent with forwardOp={type:'delete_bullet', id} and inverseOp={type:'restore_bullet', id}

**markComplete(userId, bulletId, isComplete: boolean)**:
Simple UPDATE, no undo recording (BULL-12 requirement does not list undo).

**setCollapsed(userId, bulletId, isCollapsed: boolean)**:
In transaction: UPDATE SET isCollapsed; recordUndoEvent (collapse toggle IS a structural op per CONTEXT.md).

**getDocumentBullets(userId, documentId)**:
SELECT * FROM bullets WHERE documentId=$doc AND userId=$user AND deletedAt IS NULL ORDER BY position ASC

All functions validate that the bullet belongs to the requesting userId before mutating.
