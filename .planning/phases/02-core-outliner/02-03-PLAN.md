Create `server/src/routes/bullets.ts`. Mirror the structure of `documents.ts` — Router, use(requireAuth), then routes.

**Routes to implement:**

```
GET  /documents/:docId/bullets          → getDocumentBullets(user.id, docId)          → 200 + array
POST /                                  → createBullet(user.id, body)                  → 201 + bullet
PATCH /:id                              → content|isComplete|isCollapsed PATCH         → 200 + bullet
DELETE /:id                             → softDeleteBullet(user.id, id)               → 200
POST /:id/indent                        → indentBullet(user.id, id)                   → 200 + bullet
POST /:id/outdent                       → outdentBullet(user.id, id)                  → 200 + bullet
POST /:id/move                          → moveBullet(user.id, id, body)               → 200 + bullet
DELETE /documents/:docId/completed      → hard-delete WHERE docId+userId+isComplete=true → 200 (BULL-14 — no undo)
```

Zod validation schemas:
- createBullet body: `{ documentId: z.string().uuid(), parentId: z.string().uuid().nullable().optional(), afterId: z.string().uuid().nullable().optional(), content: z.string().default('') }`
- PATCH body: `{ content: z.string().optional(), isComplete: z.boolean().optional(), isCollapsed: z.boolean().optional() }` — dispatch to correct service based on which field is present
- move body: `{ newParentId: z.string().uuid().nullable(), afterId: z.string().uuid().nullable().optional() }`

For PATCH: if `content` present → direct DB UPDATE (no undo — content debouncing makes per-keystroke undo impractical at route level; undo for text handled via separate undo recording endpoint); if `isComplete` present → markComplete; if `isCollapsed` present → setCollapsed.

Note: Text content undo recording is handled by a separate `POST /api/bullets/:id/undo-checkpoint` endpoint that the client calls explicitly after the debounce timeout. Add this stub endpoint: accepts `{ content: string }`, records undo event for text change, returns 200.

Error handling: wrap all service calls in try/catch. If service throws with message containing 'not found' or 'unauthorized', return 404/403. Cycle guard error from moveBullet → 400 with `{ error: 'Cannot move a bullet under its own descendant' }`.

Export `bulletsRouter`.

Create `server/src/routes/undo.ts`:

```typescript
import { Router } from 'express';
import { requireAuth } from '../middleware/auth.js';
import { undo, redo, getStatus } from '../services/undoService.js';

export const undoRouter = Router();
undoRouter.use(requireAuth);

undoRouter.post('/undo', async (req, res) => {
  const user = req.user as { id: string };
  const result = await undo(user.id);
  return res.json(result);
});

undoRouter.post('/redo', async (req, res) => {
  const user = req.user as { id: string };
  const result = await redo(user.id);
  return res.json(result);
});

undoRouter.get('/undo/status', async (req, res) => {
  const user = req.user as { id: string };
  const status = await getStatus(user.id);
  return res.json(status);
});
```

Then update `server/src/index.ts` to register the two new routers. Add after the existing `app.use('/api/documents', documentsRouter)` line:


The undo router mounts at `/api` (not `/api/undo`) because it has three separate endpoints with different paths: `/undo`, `/redo`, `/undo/status`.
