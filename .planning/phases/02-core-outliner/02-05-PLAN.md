**getProjectedDepth(flatItems, activeId, overId, dragOffsetX):** implement exactly per RESEARCH.md Pattern 5. Returns constrained depth (min=0, max=prevItem.depth+1).

**handleDragEnd(event):**
1. If no overId or no activeId, reset state and return
2. activeBullet = bulletMap[activeId]
3. overIndex = flatItems.findIndex(f => f.id === overId)
4. projectedDepth = getProjectedDepth(flatItems, activeId, overId, dragOffsetX)
5. Determine newParentId: find the last item before overIndex whose depth === projectedDepth - 1; if projectedDepth === 0, newParentId = null
6. Determine afterId: the item at overIndex - 1 that shares the same parentId as the new position (or null if inserting at head)
7. Call `moveBullet({ id: activeId, newParentId, afterId })`
8. Reset activeId, dragOffsetX, overId to null

**Drop indicator:** Render a thin horizontal line (2px, color #4A90E2, full-width) between BulletNode items when dragging. Compute its position from overId and projected depth. Simple approach: use a CSS `::after` pseudo-element on the item the dragged item is hovering over, OR render a dedicated `<DropIndicator>` component at the computed index in the flatItems list during drag. The indicator shows at the drop target position with paddingLeft = projectedDepth * 24 to show the depth level visually.

**DragOverlay:** Show a semi-transparent copy of the dragged bullet (BulletNode with `isDragOverlay` prop, no children rendered, opacity 0.5).

Wrap the SortableContext items list with DndContext using sensors, closestCenter collision, onDragStart (set activeId), onDragMove (set dragOffsetX from delta.x, set overId from over?.id), onDragEnd (handleDragEnd), onDragCancel (reset state).

IMPORTANT: The flatten/SortableContext approach means ALL bullets (across all depth levels) are in ONE flat SortableContext. Do NOT nest SortableContexts.
