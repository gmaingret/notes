    2. Replace the existing sensors setup (lines 86-88) with the multi-sensor pattern:
    ```tsx
    const isMobile = window.matchMedia('(max-width: 768px)').matches;
    const sensors = useSensors(
      useSensor(PointerSensor, { activationConstraint: { distance: 5 } }),
      ...(isMobile
        ? [useSensor(TouchSensor, { activationConstraint: { delay: 250, tolerance: 5 } })]
        : [])
    );
    ```

    This call is compliant with Rules of Hooks because `useSensors` and `useSensor` are called unconditionally — the spread is just array construction, not conditional hook calls. `isMobile` is computed at render time from `window.matchMedia`, which is fine in a CSR-only Vite app.

    Do NOT add `tolerance: 0` — always use `tolerance: 5` to prevent cancellation on minor finger tremor (called out in RESEARCH.md deprecated patterns).

    Do NOT change anything else in BulletTree.tsx.
