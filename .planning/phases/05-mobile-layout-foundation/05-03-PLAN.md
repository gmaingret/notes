    **2. BulletNode.tsx — bullet dot drag handle**

    Read BulletNode.tsx to find the bullet dot element (the drag handle — typically a small circle or dot element used for dnd-kit drag). Apply the pseudo-element technique to avoid layout change:

    Add `className="bullet-dot"` to the drag handle element (or its wrapper button/span — whichever is the tappable element).

    **3. index.css — bullet dot pseudo-element expansion**

    Add to the Phase 5 CSS block:
    ```css
    /* Touch target: bullet dot drag handle — expands tap area, not visual size */
    .bullet-dot {
      position: relative;
    }
    .bullet-dot::after {
      content: '';
      position: absolute;
      inset: -14px; /* expands 16px dot to 44px tap area */
    }
    ```

    If the bullet dot is already `position: relative` via existing styles, do NOT add a duplicate rule — just add the `::after` rule.

    **4. FocusToolbar.tsx — icon buttons**

    Read FocusToolbar.tsx to find the icon buttons (bold, italic, etc. formatting toolbar). Apply `minWidth: 44, minHeight: 44, display: 'flex', alignItems: 'center', justifyContent: 'center'` to the shared button style or each individual button style object.

    **5. Breadcrumb.tsx — navigation spans**

    Read Breadcrumb.tsx. The clickable ancestor spans have `role="button"` and `tabIndex={0}`. The `display: 'inline-block'` does not reliably enforce minHeight in all browsers for span elements. Switch clickable spans to `display: 'inline-flex'` with `alignItems: 'center'` and `minHeight: 44` for reliable touch targets.
