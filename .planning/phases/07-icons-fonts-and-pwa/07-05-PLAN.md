<!-- favicon.svg design guidance (from CONTEXT.md + RESEARCH.md) -->
Design: letter-mark "N" on solid background.
Background color: #1a1a1a (dark neutral — recommended by RESEARCH.md for readability on any wallpaper).
Letter color: white (#ffffff).

CRITICAL: Do NOT use a <text> element with font references in the SVG. The sharp PNG generator (librsvg) cannot resolve web fonts. Use SVG path data for the letter N, or use a <rect> background with a <text> that uses a common system font (only as a fallback for the SVG favicon in browser — but then generate PNGs differently, see below).

RECOMMENDED APPROACH: Use a <text> element with font-family="system-ui, sans-serif" for the browser favicon (browsers render this fine). For the PNG generation, use a different SVG string in the generation script that renders the N as filled rectangle paths — OR use sharp's text overlay feature if available, OR simply render the simple SVG (sharp can handle basic SVG shapes + text with system fonts on Linux).

ACTUALLY SIMPLEST APPROACH: Create the favicon SVG with a <rect> and <text> using font-family="Arial, sans-serif". Sharp on Linux uses librsvg which CAN render system fonts (Arial/Helvetica are available). Use this for both. Test the PNG output visually.

<!-- generate-icons.mjs script -->
```js
// client/scripts/generate-icons.mjs
// Run locally: node scripts/generate-icons.mjs
// Do NOT run on server — commit generated PNGs to git instead
import sharp from 'sharp';
import { resolve, dirname } from 'path';
import { fileURLToPath } from 'url';

const __dirname = dirname(fileURLToPath(import.meta.url));
const svgPath = resolve(__dirname, '../public/favicon.svg');

await sharp(svgPath).resize(192).png().toFile(resolve(__dirname, '../public/icon-192.png'));
await sharp(svgPath).resize(512).png().toFile(resolve(__dirname, '../public/icon-512.png'));
console.log('Generated: icon-192.png, icon-512.png');
```
