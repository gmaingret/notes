# Phase 7.1: UI Polish Tweaks - Context

**Gathered:** 2026-03-10
**Status:** Ready for planning

<domain>
## Phase Boundary

Four targeted UX improvements to existing UI: bullet row spacing, FocusToolbar icon replacement, Logout/Export moved to sidebar bottom, and + button auto-activates Docs tab + inline rename. No new features, no new routes, no structural refactors.

</domain>

<decisions>
## Implementation Decisions

### Bullet spacing
- Add `2px` padding-top and padding-bottom to each bullet row
- Applied via the outer row div in `BulletNode.tsx` (the div with `paddingLeft: depth * 24`)
- Keeps rows compact (Notion/Logseq density) while giving visible breathing room

### FocusToolbar icons
- Replace **all** HTML entities and Unicode chars with Lucide icons
- Standard: `size={20}`, `strokeWidth={1.5}` — consistent with Phase 7 sidebar icons
- Mapping:
  | Current | Lucide replacement |
  |---------|-------------------|
  | &#8677; indent | `<IndentIncrease />` |
  | &#8676; outdent | `<IndentDecrease />` |
  | &#8593; move up | `<ArrowUp />` |
  | &#8595; move down | `<ArrowDown />` |
  | &#8617; undo | `<Undo2 />` |
  | &#8618; redo | `<Redo2 />` |
  | &#128206; attach | `<Paperclip />` (already used in AttachmentRow) |
  | note icon | `<StickyNote />` |

### Logout & Export placement
- Remove from `...` dropdown entirely
- Add a footer section at the bottom of the sidebar `<aside>` — below the scrollable content area
- Layout: two stacked rows, each with icon + label text
  - Row 1: `<Upload />` icon + "Export all" — calls `exportAll()`
  - Row 2: `<LogOut />` icon + "Logout" — calls `handleLogout()`
- Style: subtle, muted text color (`var(--color-text-muted)`), hover shows `var(--color-bg-surface)` background
- The `...` button in the sidebar header can be removed (no remaining items in the dropdown)
- Footer sits outside the scrollable div so it's always visible

### + button behavior
- On click: (1) switch `sidebarTab` to `'docs'`, then (2) call `createDocument()`, then (3) trigger inline rename on the newly created doc row
- Inline rename: in the **sidebar DocumentList row** — the doc name becomes an editable `<input>`, auto-focused, with Enter/blur to confirm
- If sidebar is already on Docs tab, skip the tab switch (just create + rename)
- `DocumentList` needs to accept a "pending rename" doc ID from the parent (`Sidebar`) so it can render the rename input on the right row

### Claude's Discretion
- Exact CSS class names for sidebar footer
- Whether to pass `pendingRenameId` via prop or Zustand store (prefer prop — it's transient UI state local to Sidebar)
- Rename input styling (match existing doc row text style)
- Whether to remove the `...` button entirely or leave it empty (remove it — no items left)

</decisions>
