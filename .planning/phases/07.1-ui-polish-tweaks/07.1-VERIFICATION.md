### Human Verification Required

#### 1. FocusToolbar SVG icons on mobile

**Test:** Open https://notes.gregorymaingret.fr on a mobile device or in browser DevTools mobile viewport. Tap any bullet to focus it and trigger the toolbar. Inspect each of the 11 buttons.
**Expected:** All buttons display clean Lucide SVG icons (Indent, Outdent, ArrowUp, ArrowDown, Undo2, Redo2, Paperclip, StickyNote, Bookmark, Check, Trash2) — no browser-rendered Unicode glyphs.
**Why human:** Font fallback rendering is browser/OS dependent and cannot be verified by static file scan.

#### 2. Bullet row vertical breathing room

**Test:** Open any document with multiple bullet items at https://notes.gregorymaingret.fr.
**Expected:** Each row has a small but visible gap above and below — Notion/Logseq-style compact density, not crammed together.
**Why human:** 2px is confirmed in code, but whether the spacing "feels right" is an aesthetic judgment requiring visual inspection.

#### 3. Sidebar footer always-visible below scrollable list

**Test:** Open the sidebar with enough documents to require scrolling. Scroll the list to the bottom.
**Expected:** "Export all" and "Logout" rows remain fixed at the bottom of the sidebar, always visible. The ... button (three dots) is absent from the header. Hovering over footer rows shows a subtle background highlight.
**Why human:** Sticky/flex layout behavior and scroll containment require a live browser test.

#### 4. + button creates doc, switches tab, and opens inline rename

**Test:** Navigate to the Tags tab in the sidebar. Click the + button in the header.
**Expected:** (a) Docs tab activates automatically, (b) a new document row appears in the list, (c) the new row is immediately in rename mode with the cursor active in the input — no context menu or extra steps required. Type a name and press Enter to confirm.
**Why human:** The async sequence (mutate onSuccess -> setPendingRenameId -> React Query cache invalidation -> re-render -> initiallyRenaming seeds useState -> autoFocus) must be verified in a real browser. Static analysis confirms all wiring is in place but cannot simulate the timing.
