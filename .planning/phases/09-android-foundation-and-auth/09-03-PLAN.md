**EncryptedDataStoreFactory.kt** — Helper to create encrypted DataStore instances using Tink:
- Use `MasterKey.Builder(context).setKeyScheme(MasterKey.KeyScheme.AES256_GCM).build()` from `androidx.security.crypto` (if available with Tink) OR use Tink's `KeysetHandle` directly with `AeadConfig.register()` + `KeysetHandle.generateNew(AeadKeyTemplates.AES256_GCM)` to encrypt/decrypt DataStore values manually.
- Provide a factory method that returns a `DataStore<Preferences>` with encrypted storage.
- NOTE: Since DataStore Preferences doesn't have native encryption, the pragmatic approach is to use a standard `DataStore<Preferences>` and encrypt/decrypt individual string values using Tink's `Aead` primitive before storing. Create a helper: `suspend fun encryptString(value: String): String` (Base64-encoded ciphertext) and `suspend fun decryptString(value: String): String`.

**TokenStore.kt** — Wrapper around encrypted DataStore for access token:
```kotlin
class TokenStore @Inject constructor(@ApplicationContext context: Context) {
    private val dataStore = context.createDataStore("auth_tokens")
    // Or use preferencesDataStore delegate

    suspend fun getAccessToken(): String?
    suspend fun saveAccessToken(token: String)
    suspend fun clearAll()
}
```
- Encrypt the access token value using Tink before storing in DataStore preferences.
- Use `stringPreferencesKey("access_token")` as the key.
- `clearAll()` clears both the token and any cached user data.

**DataStoreCookieJar.kt** — Custom OkHttp CookieJar:
- Implements `okhttp3.CookieJar` interface.
- `saveFromResponse()`: Filter persistent cookies, serialize each as JSON string (name, value, expiresAt, domain, path, secure, httpOnly), encrypt with Tink, store in a separate `DataStore<Preferences>` keyed by `"host|name"`.
- `loadForRequest()`: Read all cookies for the request host, decrypt, filter expired ones, convert back to `okhttp3.Cookie` objects.
- Use `runBlocking` in both methods — acceptable because OkHttp calls these on its thread pool, not the main thread.
- CRITICAL: Must handle the production server's Set-Cookie format: `refreshToken=...; Path=/; HttpOnly; Secure; SameSite=Strict` (maxAge = 7 days).
- Define a `StoredCookie` data class (name, value, expiresAt, domain, path, secure, httpOnly) for serialization. Use Gson for JSON conversion.

**AuthInterceptor.kt** — OkHttp Interceptor:
- `@Inject constructor(private val tokenStore: TokenStore)`
- In `intercept()`: read access token from TokenStore via `runBlocking { tokenStore.getAccessToken() }`, if non-null add `Authorization: Bearer $token` header.
- Do NOT store token as a class field (race condition on concurrent refresh per research anti-patterns). Always read from TokenStore on each intercept call.

**TokenAuthenticator.kt** — OkHttp Authenticator with Mutex:
- `@Inject constructor(private val tokenStore: TokenStore, private val authApi: Lazy<AuthApi>)` — use `dagger.Lazy` to break the circular dependency (OkHttp -> Authenticator -> AuthApi -> OkHttp).
- Use `Mutex()` from `kotlinx.coroutines.sync`.
- In `authenticate()`:
  1. If request had no Authorization header, return null (not an auth request).
  2. `runBlocking { mutex.withLock { ... } }` — acquire lock.
  3. After acquiring lock: compare current token in TokenStore with the token from the failed request. If they differ, another coroutine already refreshed — retry with new token.
  4. If same: call `authApi.get().refresh()`, save new token via `tokenStore.saveAccessToken()`, return request with new token.
  5. On exception: `tokenStore.clearAll()`, return null (caller gets 401 — triggers logout in ViewModel).
- CRITICAL: Must handle the case where `authenticate()` is called after a previous `authenticate()` call already cleared tokens (return null immediately if no access token exists).

**AuthRepositoryImpl.kt** — Implements `AuthRepository`:
- `@Inject constructor(private val authApi: AuthApi, private val tokenStore: TokenStore)`
- `register()`: Call `authApi.register()`, save access token, map `UserDto` to domain `User`, wrap in `Result.success`. Catch exceptions -> `Result.failure`.
- `login()`: Same pattern with `authApi.login()`.
- `loginWithGoogle()`: Call `authApi.googleToken()`, same pattern.
- `refresh()`: Call `authApi.refresh()`, save token, return `Result.success(token)`.
- `logout()`: Try `authApi.logout()` (ignore failure), `tokenStore.clearAll()`.
- `getAccessToken()`: Delegate to `tokenStore.getAccessToken()`.
- `isLoggedIn()`: Return `tokenStore.getAccessToken() != null`.

**Use cases** — Each is a simple class with `operator fun invoke()`:
- `LoginUseCase(private val authRepository: AuthRepository)`: `suspend operator fun invoke(email: String, password: String): Result<User> = authRepository.login(email, password)`
- `RegisterUseCase(private val authRepository: AuthRepository)`: same pattern with register.
- `LoginWithGoogleUseCase(private val authRepository: AuthRepository)`: same with loginWithGoogle.
- `CheckAuthUseCase(private val authRepository: AuthRepository)`: `suspend operator fun invoke(): Boolean` — calls `authRepository.refresh()` to validate the refresh cookie, returns `isSuccess`. This is used by the splash screen to check cold start auth.
- `LogoutUseCase(private val authRepository: AuthRepository)`: `suspend operator fun invoke() = authRepository.logout()`.
- All use cases are `@Inject constructor`.

**NetworkModule.kt** — Hilt `@Module` `@InstallIn(SingletonComponent::class)`:

```kotlin
@Provides @Singleton
fun provideOkHttpClient(
    authInterceptor: AuthInterceptor,
    tokenAuthenticator: TokenAuthenticator,
    cookieJar: DataStoreCookieJar
): OkHttpClient = OkHttpClient.Builder()
    .addInterceptor(authInterceptor)
    .authenticator(tokenAuthenticator)
    .cookieJar(cookieJar)
    .connectTimeout(15, TimeUnit.SECONDS)  // per locked decision
    .readTimeout(15, TimeUnit.SECONDS)
    .writeTimeout(15, TimeUnit.SECONDS)
    .retryOnConnectionFailure(false)  // per locked decision — each screen handles own retry
    .build()

@Provides @Singleton
fun provideRetrofit(okHttpClient: OkHttpClient): Retrofit = Retrofit.Builder()
    .baseUrl("https://notes.gregorymaingret.fr/")  // hardcoded per locked decision
    .client(okHttpClient)
    .addConverterFactory(GsonConverterFactory.create())
    .build()

@Provides @Singleton
fun provideAuthApi(retrofit: Retrofit): AuthApi = retrofit.create(AuthApi::class.java)
```

**IMPORTANT circular dependency resolution:** TokenAuthenticator needs AuthApi, but AuthApi needs OkHttpClient which has TokenAuthenticator. Solution: inject `dagger.Lazy<AuthApi>` into TokenAuthenticator (the `@Inject constructor` for TokenAuthenticator should use `Lazy<AuthApi>`). Hilt handles this automatically with the `dagger.Lazy` wrapper.
