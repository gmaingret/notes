**AuthUiState.kt** — UI state data class:
```kotlin
data class AuthUiState(
    val selectedTab: AuthTab = AuthTab.LOGIN,
    val email: String = "",
    val password: String = "",
    val emailError: String? = null,
    val passwordError: String? = null,
    val isLoading: Boolean = false,
    val isPasswordVisible: Boolean = false,
    val isGoogleSignInAvailable: Boolean = false,
    val snackbarMessage: String? = null
)

enum class AuthTab { LOGIN, REGISTER }
```

**AuthViewModel.kt** — `@HiltViewModel`:
- Inject `LoginUseCase`, `RegisterUseCase`, `LoginWithGoogleUseCase`.
- Expose `val uiState: StateFlow<AuthUiState>` backed by `MutableStateFlow`.
- Methods:
  - `onEmailChanged(email)`: Update email, clear emailError.
  - `onPasswordChanged(password)`: Update password, clear passwordError.
  - `onTabSelected(tab)`: Switch tab, clear all errors.
  - `onPasswordVisibilityToggle()`: Toggle isPasswordVisible.
  - `onSubmit(onSuccess: () -> Unit)`:
    1. Client-side validation (submit-only, per locked decision):
       - Email: must be non-empty and match email regex -> set emailError "Enter a valid email"
       - Password: must be >= 8 chars on Register tab, non-empty on Login tab -> set passwordError "Password must be at least 8 characters"
    2. Set isLoading = true.
    3. Call `loginUseCase` or `registerUseCase` based on selectedTab.
    4. On success: invoke `onSuccess` callback.
    5. On failure: parse error response to show inline errors:
       - 409 with field "email" -> set emailError to message ("Email already registered")
       - 401 "Invalid credentials" -> set passwordError "Wrong email or password"
       - Network error -> set snackbarMessage "Can't reach server. Check your connection."
    6. Set isLoading = false.
  - `onGoogleSignIn(idToken: String, onSuccess: () -> Unit)`: Call loginWithGoogleUseCase, on failure set snackbarMessage "Google sign-in failed. Please try again."
  - `onSnackbarDismissed()`: Clear snackbarMessage.

- `val authState = MutableStateFlow(AuthState.CHECKING)`.
- In `init` block: `viewModelScope.launch { try { authState.value = if (checkAuthUseCase()) AuthState.AUTHENTICATED else AuthState.UNAUTHENTICATED } catch (e: Exception) { authState.value = AuthState.UNAUTHENTICATED } }`
- CRITICAL: Always set a terminal state — never leave as CHECKING (splash hangs per research Pitfall 5). The try/catch ensures any exception routes to UNAUTHENTICATED.

**NotesApp.kt** — Top-level composable with Navigation3:
```kotlin
@Composable
fun NotesApp(initialRoute: NavKey) {
    val backStack = rememberNavBackStack(initialRoute)
    NavDisplay(
        backStack = backStack,
        entryProvider = entryProvider {
            entry<AuthRoute> {
                AuthScreen(onAuthSuccess = { backStack.clear(); backStack.add(MainRoute) })
            }
            entry<MainRoute> {
                MainScreen(onLogout = { backStack.clear(); backStack.add(AuthRoute) })
            }
        }
    )
}
```

**MainActivity.kt** — Update to use SplashScreen and NotesApp:
- `@AndroidEntryPoint`, `@HiltAndroidApp` on Application class (already done in Plan 01).
- `private val splashViewModel: SplashViewModel by viewModels()`
- In `onCreate`:
  1. `val splashScreen = installSplashScreen()`
  2. `super.onCreate(savedInstanceState)`
  3. `splashScreen.setKeepOnScreenCondition { splashViewModel.authState.value == AuthState.CHECKING }`
  4. `enableEdgeToEdge()` for transparent system bars.
  5. `setContent { NotesTheme { val authState by splashViewModel.authState.collectAsStateWithLifecycle(); val initialRoute = if (authState == AuthState.AUTHENTICATED) MainRoute else AuthRoute; NotesApp(initialRoute = initialRoute) } }`
