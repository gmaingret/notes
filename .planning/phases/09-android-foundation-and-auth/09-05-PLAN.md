**GoogleSignInUseCase.kt** — Handles the Credential Manager two-step flow:
- `@Inject constructor(private val loginWithGoogleUseCase: LoginWithGoogleUseCase)`
- `suspend operator fun invoke(context: Context): Result<User>`:
  1. Create `CredentialManager.create(context)`.
  2. Build `GetGoogleIdOption` with `setFilterByAuthorizedAccounts(true)`, `setServerClientId(BuildConfig.GOOGLE_WEB_CLIENT_ID)`, `setAutoSelectEnabled(false)`.
  3. Try `credentialManager.getCredential(context, request)`.
  4. On `NoCredentialException`: fall back to `GetSignInWithGoogleOption.Builder(BuildConfig.GOOGLE_WEB_CLIENT_ID).build()` and retry.
  5. Extract ID token: check `credential is CustomCredential && credential.type == GoogleIdTokenCredential.TYPE_GOOGLE_ID_TOKEN_CREDENTIAL`, then `GoogleIdTokenCredential.createFrom(credential.data).idToken`.
  6. Call `loginWithGoogleUseCase(idToken)` to exchange with backend.
  7. Return the Result.
  8. Wrap all exceptions in `Result.failure`.

- Also add a companion or top-level function `isGoogleSignInAvailable(context: Context): Boolean` — try creating CredentialManager, return true. On `UnsupportedOperationException` return false. Per locked decision: hide Google button entirely if unavailable.

**Update AuthViewModel.kt:**
- Inject `GoogleSignInUseCase`.
- In `init` block (or a method called from AuthScreen): check `GoogleSignInUseCase.isGoogleSignInAvailable(context)` and update `uiState.isGoogleSignInAvailable`. Since ViewModel shouldn't hold Activity context, pass the availability check result from AuthScreen via a method `setGoogleSignInAvailability(available: Boolean)` called from a `LaunchedEffect`.
- `fun onGoogleSignIn(context: Context, onSuccess: () -> Unit)`: set loading, call `googleSignInUseCase(context)`, on success invoke callback, on failure set snackbarMessage "Google sign-in failed. Please try again." (per locked decision).

**Update AuthScreen.kt:**
- Add a `LaunchedEffect(Unit)` that checks Google Sign-In availability and calls `viewModel.setGoogleSignInAvailability(isAvailable)`.
- The Google SSO button's `onClick`: call `viewModel.onGoogleSignIn(context, onAuthSuccess)` where `context` is obtained from `LocalContext.current`.
- Button shows spinner during Google sign-in (same pattern as email/password loading per locked decision).

**IMPORTANT:** The `serverClientId` in `GetGoogleIdOption` MUST be the **Web client ID** (not Android client ID) — per research Pitfall 7. This is read from `BuildConfig.GOOGLE_WEB_CLIENT_ID` which comes from `local.properties`.

Create `.github/workflows/android-ci.yml`:

```yaml
name: Android CI

on:
  push:
    branches: ['phase-*']
    paths: ['android/**']
  pull_request:
    branches: [main]
    paths: ['android/**']

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up JDK 21
        uses: actions/setup-java@v4
        with:
          java-version: '21'
          distribution: 'temurin'

      - name: Setup Gradle
        uses: gradle/actions/setup-gradle@v4

      - name: Create local.properties
        run: |
          echo "GOOGLE_WEB_CLIENT_ID=" > android/local.properties

      - name: Run unit tests
        working-directory: android
        run: ./gradlew test

      - name: Build debug APK
        working-directory: android
        run: ./gradlew assembleDebug
```

Per locked decisions:
- Triggers on PR to main + push to phase-* branches.
- JDK 21 (locked decision).
- Creates empty `local.properties` so BuildConfig generates (GOOGLE_WEB_CLIENT_ID will be empty in CI — that's fine, SSO won't be tested in CI).
- Runs `./gradlew test` then `./gradlew assembleDebug`.
- Only triggers when `android/**` files change (don't run on server-only changes).
