  <behavior>
    MainUiState:
    - Loading: initial state, skeleton shimmer
    - Success: documents list + openDocumentId + inlineEditingDocId (nullable)
    - Error: error message string
    - Empty: no documents exist

    MainViewModel behaviors:
    - On init: loads documents via GetDocumentsUseCase, reads lastDocId, auto-opens last doc (or first if last not found, or Empty if none)
    - loadDocuments(): fetches list, transitions to Success/Error/Empty
    - openDocument(doc): sets openDocumentId, calls OpenDocumentUseCase (fire-and-forget)
    - createDocument(): calls CreateDocumentUseCase("Untitled"), adds to list, sets inlineEditingDocId to new doc ID
    - submitRename(id, title): calls RenameDocumentUseCase, updates doc in list, clears inlineEditingDocId
    - cancelRename(): clears inlineEditingDocId; if the doc being edited is a new doc with title "Untitled" that was never submitted, leave it (user can rename later)
    - deleteDocument(id): calls DeleteDocumentUseCase, removes from list; if deleted doc was open, auto-open next (or previous if last, or Empty if none remain)
    - moveDocumentLocally(fromIndex, toIndex): reorders list in-memory for optimistic UI
    - commitReorder(docId): computes afterId from current list order (the doc above docId, or null if first), calls ReorderDocumentUseCase; on failure, revert list and show snackbar
    - refreshDocuments(): called when drawer opens -- reloads from API
  </behavior>
  <action>
    1. Create `presentation/main/MainUiState.kt`:
       ```kotlin
       sealed interface MainUiState {
           data object Loading : MainUiState
           data class Success(

               val openDocumentId: String?,
               val inlineEditingDocId: String? = null
           ) : MainUiState
           data class Error(val message: String) : MainUiState
           data object Empty : MainUiState
       }
       ```

       - `moveDocumentLocally(fromIndex, toIndex)`: update the documents list in-place. Use `toMutableList().apply { add(toIndex, removeAt(fromIndex)) }`
       - `commitReorder(docId)`: compute afterId by finding the document at `currentIndex - 1` in the current list (null if index == 0). Call ReorderDocumentUseCase. On failure: reload documents from API (revert) and emit snackbar message.
       - `loadDocuments()` cold-start logic: fetch list, if empty -> Empty state. Otherwise read lastDocId. If lastDocId found in list -> open it. If not found (deleted or first launch) -> open first doc. Save the opened doc's ID.
       - `refreshDocuments()`: fetch list and update Success state but preserve the current openDocumentId. If current open doc is no longer in list (deleted elsewhere), open first doc.
