    Replace all stub methods in BulletTreeViewModel with full implementations.

    **Debounce implementation for content and note:**
    Use a `MutableSharedFlow<Pair<String, String>>` for content changes and another for note changes. In init{}, collect each with `debounce(500)` and fire the PATCH. This avoids per-keystroke API calls. Keep a local `contentOverrides: MutableMap<String, String>` so the UI reflects the latest typed text even before debounce fires.

    **Optimistic update pattern** (same for all structural operations):
    1. Read current bullets from Success state
    2. Apply local mutation to bullets list (change parentId, recompute position, etc.)
    3. Re-run flattenTreeUseCase and emit new Success state
    4. Enqueue the API call
    5. On API success: update the specific bullet with server response (authoritative position/parentId)
    6. On API failure: emit snackbar, call reloadFromServer()

    **createBullet specifics:**
    - Optimistic: generate a temporary UUID, insert bullet after afterBulletId in the local list
    - API: POST /api/bullets with {documentId, parentId, afterId, content: ""}
    - On success: replace temp bullet with server-returned bullet (real ID), set focusedBulletId to real ID

    **enterOnEmpty specifics:**
    - Find the bullet by ID. If it has a parentId and content is empty, call outdentBullet instead
    - If root-level and empty, just clear focus (setFocusedBullet(null))

    **backspaceOnEmpty specifics:**
    - Find previous visible bullet in flatList (index - 1)
    - If deleted bullet has children: reparent them to the previous bullet's parent (or same parent as deleted bullet)
    - Optimistic: remove bullet from list, reparent children, re-flatten
    - Set focus to previous bullet with focusCursorEnd=true
    - Enqueue: DELETE /api/bullets/:id

    **moveUp/moveDown with cross-parent:**
    - moveUp: if bullet is first sibling, target = parent's previous sibling's last child position. Otherwise target = previous sibling position.
    - moveDown: if bullet is last sibling, target = parent's next sibling's first child position. Otherwise target = next sibling position.
    - Use moveBulletUseCase with computed newParentId and afterId.

    **zoomTo specifics:**
    - Set _zoomRootId to bulletId (null = back to document root)
    - Build breadcrumb: walk up parentId chain from bulletId to document root, collect bullets, reverse
    - Re-flatten with rootId parameter
