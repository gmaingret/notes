    **BulletMarkdownRenderer.kt** â€” two public functions:

    1. `fun buildMarkdownAnnotatedString(text: String): AnnotatedString`
       - Parse markdown patterns and build AnnotatedString with SpanStyles
       - Bold: `\*\*(.+?)\*\*` -> fontWeight=Bold
       - Italic: `\*(.+?)\*` or `_(.+?)_` -> fontStyle=Italic (must not conflict with bold â€” process bold first, then italic on remaining non-bold regions)
       - Strikethrough: `~~(.+?)~~` -> textDecoration=LineThrough
       - Links: `\[(.+?)\]\((.+?)\)` -> color=Blue, textDecoration=Underline, add LinkAnnotation.Url
       - Process patterns by finding all match ranges, sorting by start position, applying SpanStyles

    2. `fun parseContentSegments(text: String): List<ContentSegment>`
       - Split text into TextSegment and ChipSegment objects for composable rendering
       - ChipSegment types: TAG (#word), MENTION (@word), DATE (!!yyyy-mm-dd or !!word)
       - Regex: `(#\w+)|(@\w+)|(!![\w-]+)`
       - Returns ordered list of segments preserving original text order

    ```kotlin
    sealed interface ContentSegment {
        data class TextSegment(val text: String) : ContentSegment
        data class ChipSegment(val type: ChipType, val text: String) : ContentSegment
    }
    enum class ChipType { TAG, MENTION, DATE }
    ```

    **Enter detection via onValueChange (primary â€” software keyboard reliability):**
    ```kotlin
    onValueChange = { newValue ->
        val newText = newValue.text
        val newlineIndex = newText.indexOf('\n')
        if (newlineIndex >= 0) {
            val textBeforeNewline = newText.substring(0, newlineIndex)
            if (textBeforeNewline.isEmpty() && localText.isEmpty()) {
                onEnterOnEmpty()
            } else {
                onContentChange(textBeforeNewline)
                onEnterWithContent()
            }
            // Don't update TextFieldValue with the newline
        } else {
            localText = newText
            textFieldValue = newValue
            onContentChange(newText)
        }
    }
    ```

    **Backspace on empty via onValueChange:** Cannot detect via onValueChange (text is already empty). Use `onKeyEvent` for Key.Backspace when text is empty. For software keyboards that don't send key events, use a sentinel character approach: if text is empty, insert a zero-width space and detect its deletion. OR simpler: rely on `onKeyEvent` for Backspace (works on most keyboards including Gboard for delete-backward). If Backspace detection proves unreliable, the toolbar can provide a dedicated delete button as fallback.
