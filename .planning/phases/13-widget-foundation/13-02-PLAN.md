Create `WidgetConfigViewModel.kt` as a `@HiltViewModel` with:

1. **State**: Define a `ConfigUiState` sealed interface with: `Loading`, `NeedsLogin`, `DocumentsLoaded(documents: List<Document>)`, `Error(message: String)`.
2. **Dependencies** (injected): `DocumentRepository`, `AuthRepository` (for checkAuth and login), `TokenStore` (to check if access token exists).
3. **Init block**: Launch coroutine to check auth. If `tokenStore.getAccessToken() != null`, try `authRepository.checkAuth()`. On success, call `loadDocuments()`. On failure (token expired), set state to NeedsLogin. If no token, set state to NeedsLogin.
4. **loadDocuments()**: Calls `documentRepository.getDocuments()`. On success with non-empty list, emit `DocumentsLoaded`. On failure, emit `Error`.
5. **login(email, password)**: Calls `authRepository.login()`. On success, calls `loadDocuments()`. On failure, emits error.
6. **loginWithGoogle(idToken)**: Calls `authRepository.loginWithGoogle()`. On success, calls `loadDocuments()`. On failure, emits error.
7. **selectDocument(appWidgetId, docId)**: Saves via `WidgetStateStore.saveDocumentId(appWidgetId, docId)`. Emits a `DocumentSelected` one-shot event (use `Channel<Unit>` or SharedFlow).

Note: WidgetConfigViewModel is @HiltViewModel — it CAN use @Inject constructor because the config Activity IS a ComponentActivity annotated with @AndroidEntryPoint.

Create `WidgetConfigActivity.kt` as a `@AndroidEntryPoint ComponentActivity` with `setContent`:

1. **CRITICAL first line in onCreate**: Extract `appWidgetId` from `intent.extras` (EXTRA_APPWIDGET_ID). If INVALID_APPWIDGET_ID, finish(). Then immediately call `setResult(RESULT_CANCELED, Intent().putExtra(EXTRA_APPWIDGET_ID, appWidgetId))`. This prevents the launcher from silently discarding the widget on back press.

2. **Compose UI in setContent**: Use `NotesTheme` (the app's existing Material 3 theme). Observe `viewModel.uiState` via `collectAsState()`.

3. **NeedsLogin state**: Show a login form with email/password fields and a login button. Add a Google SSO button using the existing Credential Manager pattern from the app's AuthScreen. On successful login, the ViewModel automatically loads documents.

4. **DocumentsLoaded state**: Show a `LazyColumn` with document names. Each item is a clickable row showing just the document title (no metadata). Tapping a document:
   - Calls `viewModel.selectDocument(appWidgetId, doc.id)`
   - Calls `NotesWidget().updateAll(context)` to trigger widget re-render
   - Sets `setResult(RESULT_OK, Intent().putExtra(EXTRA_APPWIDGET_ID, appWidgetId))`
   - Calls `finish()`

5. **Loading state**: Show a centered `CircularProgressIndicator`.

6. **Error state**: Show error message with a retry button.

7. Collect the `DocumentSelected` event from ViewModel to trigger the RESULT_OK + finish flow.

8. The Activity theme should be `@style/Theme.Notes` (already registered in manifest from Plan 01). Use `enableEdgeToEdge()` for modern system bars handling.

Note on Google SSO in config activity: The config activity is a full ComponentActivity so Credential Manager works identically to the main app's auth screen. Reuse the same `GetGoogleIdOption` configuration with the web client ID from BuildConfig or resources.
