    **1. Add file type allowlist to multer config in `server/src/routes/attachments.ts`:**

    Create a Set of allowed extensions:
    ```
    ALLOWED_EXTENSIONS = new Set([
      '.jpg', '.jpeg', '.png', '.gif', '.webp', '.avif', '.svg',
      '.pdf', '.txt', '.csv', '.json', '.xml', '.md',
      '.docx', '.xlsx', '.pptx',
      '.zip', '.gz', '.tar'
    ])
    ```

    Add a `fileFilter` callback to the multer config that:
    - Extracts the file extension from `file.originalname` (lowercase)
    - If extension is in ALLOWED_EXTENSIONS, call `cb(null, true)`
    - Otherwise, call `cb(new Error('File type .xyz is not allowed. Supported: images, PDFs, documents, archives'))` — per user decision on error message format

    In the multer error handler middleware (lines 46-52), add a check for this custom error to return 400 with the error message.

    **2. Add bullet ownership verification in `server/src/services/attachmentService.ts`:**

    Add a new exported function:
    ```typescript
    export async function verifyBulletOwnership(userId: string, bulletId: string): Promise<boolean> {
      const [bullet] = await db
        .select({ id: bullets.id })
        .from(bullets)
        .where(and(eq(bullets.id, bulletId), eq(bullets.userId, userId)));
      return !!bullet;
    }
    ```

    Import `bullets` from schema. This follows the existing pattern in bulletService.ts.

    **3. Apply ownership checks in attachment routes:**

    Per user decision, return 404 (not 403) when bullet is not owned by user — hides bullet existence.

    - POST `/bullets/:bulletId`: After multer processes the file, call `verifyBulletOwnership(user.id, bulletId)`. If false, delete the uploaded file (prevent orphan) and return `404 { error: 'Bullet not found' }`.
    - GET `/bullets/:bulletId` (list): Call `verifyBulletOwnership(user.id, bulletId)` before querying. If false, return `404 { error: 'Bullet not found' }`.

    Note: GET `/:id/file` and DELETE `/:id` already filter by userId on the attachments table, so they already enforce ownership at the attachment level. No change needed.

    **4. Create tests in `server/tests/attachments.test.ts`:**

    Add tests for:
    - verifyBulletOwnership returns true when bullet belongs to user
    - verifyBulletOwnership returns false when bullet does not belong to user
    - (The fileFilter behavior is best tested via integration/E2E, but unit test the ALLOWED_EXTENSIONS set if exported)
