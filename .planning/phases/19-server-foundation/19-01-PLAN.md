**undo.ts changes:**

1. Remove all three dead `try { ... } catch (err) { throw err; }` wrappers — Express 5 handles async throws natively.

2. For POST /undo handler: Before calling `undo(db, user.id)`, call `getStatus(db, user.id)` to check `canUndo`. If `canUndo === false`, return `res.status(422).json({ error: 'Nothing to undo' })`. Otherwise call `undo(db, user.id)` and return the result. The undo service's `undo()` function silently returns status when stack is empty (doesn't throw), so the route must pre-check.

3. For POST /redo handler: Same pattern — call `getStatus(db, user.id)` first, if `canRedo === false` return `res.status(422).json({ error: 'Nothing to redo' })`. Otherwise call `redo(db, user.id)` and return.

4. For GET /undo/status handler: Simply `const status = await getStatus(db, user.id); return res.json(status);` — no try/catch needed.

**index.ts changes:**

```typescript
// Global error handler — catches unhandled errors from all routes
// Must have 4 parameters for Express to recognize it as error middleware
app.use((err: Error, _req: express.Request, res: express.Response, _next: express.NextFunction) => {
  console.error(err);
  res.status(500).json({ error: 'Internal server error' });
});
```
