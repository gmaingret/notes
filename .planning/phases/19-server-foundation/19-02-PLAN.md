Create `.github/workflows/server-ci.yml` with the following exact content:

```yaml
name: Server CI

on:
  push:
    branches: ['phase-*']
    paths: ['server/**']
  pull_request:
    branches: [main]
    paths: ['server/**']

jobs:
  test:
    runs-on: ubuntu-latest

    services:
      postgres:
        image: postgres:17-alpine
        env:
          POSTGRES_DB: notes_test
          POSTGRES_USER: notes_user
          POSTGRES_PASSWORD: testpassword
        ports:
          - 5432:5432
        options: >-
          --health-cmd "pg_isready -U notes_user"
          --health-interval 5s
          --health-timeout 5s
          --health-retries 10

    env:
      DATABASE_URL: postgresql://notes_user:testpassword@localhost:5432/notes_test
      JWT_SECRET: ci-test-jwt-secret-not-real
      JWT_REFRESH_SECRET: ci-test-jwt-refresh-secret-not-real
      NODE_ENV: test

    steps:
      - uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: 'npm'
          cache-dependency-path: server/package-lock.json

      - name: Install dependencies
        working-directory: server
        run: npm ci

      - name: Build
        working-directory: server
        run: npm run build

      - name: Run tests
        working-directory: server
        run: npm test
```

Key decisions:
- Node.js 22 (current LTS, matches server development)
- Postgres 17-alpine service container (matches docker-compose.yml)
- Test-specific DATABASE_URL pointing to localhost (CI maps container port to host)
- Fake JWT secrets — CI only validates code, never runs in production
- No SSH deploy step per CLAUDE.md — CI is validation-only
- No lint step — server/package.json has no lint script (Claude's discretion: skip rather than add one in this plan)

      - name: Lint
        working-directory: client
        run: npm run lint

      - name: Build
        working-directory: client
        run: npm run build

Key decisions:
- No Postgres needed — client has no database dependency

- `npm run build` runs both `tsc -b` and `vite build` (as defined in client/package.json)
- No SSH deploy step per CLAUDE.md
- Node.js 22 matching server CI
